# Solace Branding Guidelines

Comprehensive brand identity skill for ensuring Solace visual and verbal consistency. Based on the official Solace Brand Book Version 3.0 (2025).

## When to Use

- **Visual Asset Creation**: Designing presentations, social media, marketing materials
- **Color Selection**: Choosing appropriate color palettes for different modes
- **Typography**: Selecting fonts and establishing type hierarchy
- **Logo Usage**: Ensuring correct logo placement and sizing
- **Brand Compliance Review**: Auditing materials for brand consistency
- **Illustration Guidelines**: Creating diagrams, icons, and illustrations
- **Photography Direction**: Selecting and treating images

> **Note**: Contact Solace for permission before using logos. For Deep Blue variant or additional formats (PNG, EPS), contact press@solace.com or visit the [Press Center](https://solace.com/press-center/).

#### Logo Design
The Solace logotype is derived from characters in the ITC Souvenir font. The letterforms for "s," "a," and "e" were custom-stylized to align with the unique visual aesthetic rooted in the Solace brand's heritage.

#### Acceptable Logo Colors
The logotype should always be used as a solid color:

| Color | Hex | Usage |
|-------|-----|-------|
| **Deep Blue** | `#093B5F` | Primary on light backgrounds |
| **Classic Green** | `#00C895` | On dark backgrounds |
| **White** | `#FFFFFF` | On dark backgrounds |

#### Logo Clear Space
The necessary clear space around the logo is based on **1.5x the height of the lowercase "s"** in the logotype, ensuring optimal visibility and legibility.

#### Minimum Size
- **Digital**: 70px wide minimum
- **Print**: 25mm wide minimum

#### Logo Usage Rules

✓ **DO:**
- Keep the logo integrity and form as designed
- Use only approved colors (Deep Blue, Classic Green, White)
- Find an area with high contrast for logo placement over photos
- Create high contrast space with branded shapes for logo placement

✗ **DON'T:**
- Change the logotype font
- Stretch the logo—always maintain designed proportions
- Outline the logo
- Change logo colors to unapproved colors
- Use styles like drop shadows or glow effects
- Overlap logo with busy areas of photos
- Bleed logo out of branded shape

#### Solace "S" Icon Usage
For smaller formats (favicons, social icons, app icons):

| Format | Modes Available |
|--------|-----------------|
| **Favicon** | Light, Bright, Dark |
| **Circle Icon** | For social media profiles |
| **Square Icon** | For app icons |

#### Co-branding
When using alongside partner logos:
- Both logos should be of equal scale
- Separated by a thin vertical line stroke
- Ensure enough space around logos to stand independently
- When possible, opt for one-color lockup

| Category | Options |

> **Note**: Developer tools and code should always be displayed in dark mode.

### Color Usage Rules

✓ **DO:**
- Pair secondary colors with a primary green
- Use high contrast combinations for accessibility
- Follow WCAG accessibility guidelines

✗ **DON'T:**
- Combine two or more secondary colors together
- Use entirely monotone artworks (unless green palette from secondary)
- Use only pastels/lighter shades together without primary palette colors
- Display copy using secondary colors

### Accessibility Guidelines (WCAG)

| Level | Requirement | Ratio |
|-------|-------------|-------|
| **AA Text** | Regular text | 4.5:1 |
| **AA Text** | Large text (18pt or 14pt bold) | 3:1 |
| **AA Non-Text** | UI components & graphics | 3:1 |
| **AAA Text** | Regular text | 7:1 |
| **AAA Text** | Large text | 4.5:1 |

### Typography

#### Primary Font: New Spirit

**Usage**: H1 and H2 headlines only

| Weight | Use Case |
|--------|----------|
| **Light** | Primary headlines (preferred) |
| **Bold** | Emphasis in headlines |

**Guidelines**:
- Reserved for primary headlines
- Shines at larger sizes, especially in light weight
- Use title case—avoid all caps
- This serif brings warmth and character—let it breathe

#### Secondary Font: Figtree

**Usage**: Secondary headlines, body copy, UI elements

| Weight | Use Case |
|--------|----------|
| **Regular** | Body copy, sub-headlines |
| **Bold** | H3-H5 headlines, emphasis |

**Guidelines**:
- Modern sans serif for versatility
- All body copy should be set in Figtree
- Full range of weights for different use cases
- If it's not a headline, it belongs in Figtree
- Includes buttons, labels, secondary headings

**System Alternatives**: Segoe UI (Microsoft Word/PPT)

#### Tertiary Font: Space Mono

**Usage**: Eyebrows, code snippets, compact UI elements

| Weight | Use Case |
|--------|----------|
| **Regular** | Code snippets, overlines |
| **Bold** | Emphasis in code |

**Guidelines**:
- Keep at 16pt or smaller
- Not meant for headlines or large-scale text
- Best for small, specific bits of information
- Overline text: All caps, Solace light green on dark bg, Deep Blue on light bg

**System Alternatives**: Cascadia Mono, Aptos Mono (Microsoft)

#### Type Hierarchy

| Element | Font | Weight | Notes |
|---------|------|--------|-------|
| **Overline** | Space Mono | Regular | All caps, green on dark, Deep Blue on light |
| **H1, H2** | New Spirit | Light | Primary headlines |
| **H3, H4, H5** | Figtree | Bold | Secondary headlines |
| **Sub-headline** | Figtree | Regular/Bold | Supporting headline text |
| **Body** | Figtree | Regular/Bold | Main content |
| **Code** | Space Mono | Regular | Code snippets only |

## Graphic Language

### Brand Shapes & Patterns

Brand shapes and patterns are inspired by the idea of data and its movement. They are a core part of the visual identity and serve as a unifying, grounding element across all brand touchpoints.

#### Shape Types

| Type | Represents | Usage |
|------|------------|-------|
| **Solid Fill** | Data | Primary visual elements |
| **Outline** | Solace platform/solution | Secondary elements, no more than 3x on hero illustration |

#### Shape Usage Rules

1. ✓ Shapes can overlap with text as long as everything remains clear and readable
2. ✓ They can be scaled, cropped, or positioned to help tie different elements together
3. ✓ Great for grounding illustrations or framing photography
4. ✗ Don't bleed logo out of branded shape

### Iconography

Simple line icons with a splash of vivid green:
- Effectively communicate complex ideas quickly
- Easily recognizable across diverse audiences
- Create a cohesive visual language
- Reinforce messaging and enhance brand recall
- Seamlessly integrate into various marketing materials

### Diagrams

Architecture diagrams are important at Solace. Follow these standards:

#### Diagram Elements
- **Shapes & colors**: Use brand shape library
- **Line styles**: Consistent line work in Deep Blue (1px weight)
- **Icons**: Follow icon color usage guidelines

#### Chart Guidelines
- Depth and emphasis take priority over color
- Color should highlight key focal points
- Content should remain clean and straightforward

### Illustrations

#### Style Breakdown

| Element | Specification |
|---------|--------------|
| **Brand Shape Background** | Used as background element for hero illustrations |
| **Solace Mesh** | Apply "Mesh" background pattern to brand shapes when appropriate |
| **Outline Brand Shapes** | Represent data; use no more than 3x on hero illustration |
| **Line Work** | Always Deep Blue, 1px weight |
| **Graphic Elements** | Line/shadow in Deep Blue, fill in White, 1px weight |
| **Shadows** | Always in Deep Blue; bold and intentional; flat style |

#### Illustration Types

| Type | Usage | Examples |
|------|-------|----------|
| **Hero** | Larger story, more space, stand-alone | Website banners, PPT covers, social posts |
| **Thumbnail** | Summarized story, lighter weight | Web cards, smaller social posts |

### Photography

#### Direction
**Purposeful. Consistent. Human.**

Photography style is intentional and restrained:
- High-contrast black and white images for consistency
- Color introduced through brand backgrounds or duotone overlays
- Clear visual signature

#### User/Workplace Photos

✓ **DO:**
- Keep it real—choose candid, natural moments
- Show true tech settings (laptops, code, whiteboards)
- Reflect diversity (backgrounds, genders, ages, abilities)
- Capture genuine emotion (focus, passion, curiosity)

✗ **DON'T:**
- Stiff, overly posed shots
- Overly staged or futuristic scenes
- Cliché hacker shots, cheesy handshakes, glowing code
- Generic office shots without context

#### Industry Photos

✓ **DO:**
- Show people in action—actively working
- Provide recognizable industry context
- Include element of data or technology
- Keep it clear, not conceptual
- Use primarily in case studies and customer success stories

✗ **DON'T:**
- Environments without people
- Generic office shots
- Abstract visuals (floating graphs, symbolic icons)

#### Photo Integration Options

| Method | Description |
|--------|-------------|
| **Shape Mask** | Image masked within a brand shape |
| **Background Layer** | Full image as background with shape overlay |
| **Standalone** | Photo in foreground without masking/shapes |
| **Dynamic Outline** | Overlaid shape for motion, emphasis, or continuity |

## Brand Applications

### Website
- Follow all identity guidelines
- Dark mode for developer/code sections
- Use brand shapes to frame content

### Social Media
- LinkedIn carousel banners with consistent brand elements
- Use appropriate mode (Light/Bright/Dark) based on content
- Include dynamic outline shapes for visual continuity

### Presentations (PowerPoint)
- Use brand template
- Hero illustrations for cover slides
- Consistent typography hierarchy

### Collateral (Datasheets, Whitepapers)
- Follow type hierarchy
- Use brand shapes for visual interest
- Maintain WCAG accessibility in text/background combinations

### Events & Trade Shows
- Large-scale applications of brand shapes
- High-contrast for visibility
- Consistent logo placement

## Review Checklist

### Quick Brand Compliance Check

```markdown
## Logo
- [ ] Using approved color (Deep Blue, Classic Green, or White)
- [ ] Proper clear space maintained (1.5x "s" height)
- [ ] Minimum size met (70px/25mm)
- [ ] No stretching, outlining, or effects applied
- [ ] High contrast placement

## Colors
- [ ] Using approved brand colors
- [ ] Not combining multiple secondary colors
- [ ] WCAG accessibility ratios met
- [ ] Appropriate mode selected (Light/Bright/Dark)

## Typography
- [ ] New Spirit for H1/H2 only
- [ ] Figtree for body and secondary headlines
- [ ] Space Mono for code/overlines only (≤16pt)
- [ ] Correct weights used

## Graphics
- [ ] Brand shapes used appropriately
- [ ] Line work in Deep Blue at 1px
- [ ] Outline shapes limited to 3x on hero illustrations
- [ ] Icons use consistent line style with green accent

## Photography
- [ ] High contrast B&W treatment
- [ ] Authentic, candid imagery
- [ ] Diverse representation
- [ ] Appropriate context shown
```
